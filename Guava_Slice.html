Coding Principles

  1. High modularity & Highly layered

    2. This is good for the eyes & mind of the coder and also the system.
    We never want a monolithic style code.

  2. Max 60 lines

    3. There shouldn't be more than 50-60 lines in a particular script if it excedes we should extend it into a new script.

  3. High Cohession & Low coupling

    4. Example: We require a particular function from a set of modules A,B,C,D and we call module A.Module A finds it in C and brings back the info.
    That's the correct approach.
    If it brought all the modules instead it's a bad design and we call it coupled. It often makes a laggy game

  4. Except class names Follow Camel Case and use proper variable names

    5. public string inventoryBasics;
    public class Animal{.......};

    It makes it easy for your collborators to understand your code & contribute and is very essential for long term maintenance.
    This convention is also adhered to in most languages like Java/C#.

  5. private variables start with _

    6. private int _playerHealth = 0;

  6. Avoid braces for single line loops/Conditionals

    7. while(inventoryList--)
        cout << inventoryList << endl;

  7. Always write a short comment of what something performs

    8. //disablesPanicMusic when idle
    if(!attacked)
        disablePanicMusic();

  8. Write a short doc in the beginning of the class regarding what it performs

    9. /*
    Date: xx-xx-20xx
    @author: Abhisht,DeepKaran,Aakash
    The class is responsible for animal movement and all sounds & animations related to it.
    (A shorthand Algo) [optional]
    ....
    ....
    ....
    */
